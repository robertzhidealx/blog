I'm a member of the [JHU Programming Languages Laboratory](https://pl.cs.jhu.edu), working with [Professor Scott Smith](https://www.cs.jhu.edu/~scott)
on demand-driven execution techniques.

  - DDE lazily evaluates a program by working backwards from a specified program
    point to find its corresponding value. This theory elaborates on the essence of
    Demand-Driven Symbolic Execution ([ICFP â€™20](https://pl.cs.jhu.edu/projects/demand-driven-symbolic-execution)).

  - Create an interpreter for DDE in OCaml, with initial support for the pure lambda
    calculus and is extended to additional language constructs. Create test suites
    to benchmark DDE against forward execution techniques in terms of correctness and
    performance.
